This keeps section pages branded while letting content pages own their titles.

## 5. Sitemap & Robots

Generated by `frontend/scripts/generate-content.mjs` during `npm run prebuild` / `npm run build`:

### 5.1 Generation

- **Inputs**: Static JSON output under `frontend/public/api/`:
  - `posts.json` (blog posts)
  - `projects.json` (projects + experimental)
  - plus known static routes.
- **Outputs**:
  - `frontend/public/sitemap.xml`
  - `frontend/public/robots.txt`

### 5.2 Sitemap Contents

- **Base URL**:
  - Default: `https://keerthan.tinkernlabs.com`
  - Overrides: set `SITEMAP_BASE_URL` in the environment if you ever change the primary domain.
- **Included URLs**:
  - Static routes:
    - `/`
    - `/projects`
    - `/experimental`
    - `/blog`
    - `/cases`
    - `/resume`
    - `/about`
    - `/contact`
    - `/life`
  - Blog posts:
    - `/blog/{id}` for each post in `posts.json`
    - `<lastmod>` from `post.date` when present.
  - Projects:
    - `/projects/{id}` for each item in `projects.json` where `kind: "project"`.
  - Experimental:
    - `/experimental/{id}` where `kind: "experimental"`.

### 5.3 Robots.txt

Generated as:

```text
User-agent: *
Allow: /

Sitemap: https://keerthan.tinkernlabs.com/sitemap.xml
```

You can extend this file later (e.g., to block specific paths) by editing the generator.

## 6. Content Guidelines for Better SERPs

- **Headings**:
  - Ensure H1 on detail pages matches the main title (already true for Blog/Project detail).
  - Use meaningful H2/H3s; avoid keyword stuffing.
- **Excerpts**:
  - For blog posts, keep `excerpt` in frontmatter as a strong 1â€“2 line summary; this is used in lists and often becomes the search snippet.
  - For projects, keep `description` concise, impact-focused.
- **Keywords**:
  - Include `Keerthan Venkata` and `TinKern Labs` naturally in introductory paragraphs where relevant (Home, About, Projects, Blog intros).
  - Avoid forcing them into every heading; prioritize readability.
- **Internal Linking**:
  - Keep cross-links between related posts/projects/cases via `relatedPosts`, `relatedProjects`, `projectIds`, and (optionally) `launchpadUrl`.
  - This helps crawlers understand relationships between content and improves navigation.

## 7. How to Update SEO Safely

1. **Change role/branding**:
   - Update hero subtitle in `HomePage` (`App.tsx`).
   - Update meta title/description/keywords in `frontend/index.html`.
   - Optionally update JSON-LD `jobTitle` and `description`.
2. **Add new content**:
   - Edit `backend/app/content/` (posts, projects, experimental, cases, launchpad).
   - Run `cd frontend && npm run build` to regenerate `/api` JSON, `sitemap.xml`, and `robots.txt`.
3. **Verify**:
   - Open `https://keerthan.tinkernlabs.com/sitemap.xml` and `robots.txt` in the browser after deploy.
   - Use Google Search Console / URL Inspection to request reindexing for important pages when needed.
